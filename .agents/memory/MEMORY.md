- [Fly.io deploy gotchas](fly-io-deploy.md) — Fly needs billing info before first app create; it does not auto-inject PORT, set it explicitly in fly.toml [env].
- [Git push auth vs GitHub connector](git-push-vs-github-connector.md) — reconnecting the generic GitHub connector does not fix git push auth; use a PAT for direct pushes when the Replit git-remote credential is broken.
- [pnpm monorepo Docker builds](pnpm-monorepo-docker.md) — workspace libs exported as ./src/*.ts get bundled by esbuild directly from source, so Docker only needs full repo context + pnpm install + build; no separate lib build stage needed.
- [Next.js on a Replit web artifact](nextjs-on-replit-artifact.md) — no native nextjs artifact kind; reuse "web" kind unchanged, static-export into existing dist/public, set allowedDevOrigins for the preview proxy.
- [Replit connectors break off-platform deploys](replit-connectors-off-platform.md) — code using @replit/connectors-sdk (e.g. Resend) only works inside Replit; deployed servers (Fly/etc.) need a direct API-key fallback.
- [esbuild externals need node_modules in Docker runtime](esbuild-externals-docker-runtime.md) — externalized packages (build.mjs `external` list) aren't in the bundle; the Docker runtime stage must copy the relevant pnpm node_modules (root store + workspace symlinks) or the container crashes with ERR_MODULE_NOT_FOUND.
- [Sanity Studio in a pnpm monorepo](sanity-studio-monorepo.md) — deploy Studio v6+ standalone/hosted (not embedded) to dodge a decompress firewall block; scope React version overrides per-package, not root-wide.
- [Sanity client vs image-url in Next.js](sanity-client-vs-image-url-nextjs.md) — never import the Sanity data client into a 'use client' file; split image-url into its own client-safe module with NEXT_PUBLIC_ env vars.
